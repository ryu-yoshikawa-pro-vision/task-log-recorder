# Quickstart

1. `AGENTS.md` を読む。
2. `docs/PROJECT_CONTEXT.md` をそのプロジェクト用に更新する。
3. 最初の依頼で `.codex/runs/<run_id>/` を作る。
4. 計画系は `PLANS.md`、レビュー系は `CODE_REVIEW.md` から対応 skill を開く。
5. `bash scripts/verify` または `powershell -ExecutionPolicy Bypass -File scripts/verify.ps1` を実行して最低限のセットアップを確認する。

## 最初に編集すること
- プロジェクトの目的
- ディレクトリ構造
- 品質ゲート
- 安全制約
