# 履歴編集モーダルの「種別 / タスク名」幅比率調整

## Objective
- 履歴タブのログ編集モーダルで、先頭行の `種別` を狭く、`タスク名` を広く見せる。

## Scope
- In:
  - `chrome-extension/popup.html` の編集モーダル先頭行へ専用クラスを付与する。
  - `chrome-extension/styles.css` に編集モーダル専用の `25:75` レイアウトを追加する。
- Out:
  - `popup.js` の保存処理、DOM id、ログ構造、他画面のフォーム共通スタイル変更。

## Definition of Done
- 編集モーダル先頭行がデスクトップ幅で `25:75` の見た目になる。
- `メモ`、`タイトル`、`URL` の全幅レイアウトが維持される。
- `820px` 以下では既存どおり 1 列に戻る。
- `npm run lint` が成功する。

## Assumptions
- 要望は視覚上の幅比率変更を指し、項目順や保存仕様の変更は不要。
- 比率は `25:75` を採用する。

## Changes
- 編集モーダルのグリッドに専用クラスを追加する。
- `種別` と `タスク名` のフィールドに専用クラスを追加する。
- 専用クラスにのみ `grid-template-columns: minmax(0, 1fr) minmax(0, 3fr);` を適用する。

## Validation
- `npm run lint`
- 必要に応じて手動で編集モーダルを開き、横幅とレスポンシブ崩れがないことを確認する。
