# ポップアップ履歴UIの開閉・統計・ページネーション調整

## Summary
- 記録タブの「現在のページ」セクションを、初期状態が閉じたエクスパンションパネルに変更する。
- 履歴タブの総件数・最新の日付・表示範囲は、独立カードではなく以前のように横並びのインライン表示へ戻す。
- ページネーションは履歴リストの上部だけでなく下部にも同じ操作系を配置し、どちらからでも同じページ状態を操作できるようにする。

## Key Changes
- `chrome-extension/popup.html`
  - 記録タブの現在ページ表示を、ヘッダー押下で開閉するパネル構造へ変更する。
  - 初期表示は閉じる。閉じた状態でも「現在のページ」見出しと開閉トリガーは見える構成にする。
  - 履歴タブの 3 つの metric card を廃止し、履歴パネル上部に `総件数 / 最新の日付 / 表示範囲` を横並びで置く。
  - 既存の上部ページネーションを保持しつつ、`recentLogs` の後ろにも同じ prev / page label / next を追加する。
- `chrome-extension/popup.js`
  - 現在ページパネル用に `isPageSectionOpen` 相当の UI state を追加し、初期値は `false` にする。
  - 開閉トリガーの click と必要な `aria-expanded` / `hidden` 切り替えを実装する。
  - 履歴描画を「統計表示更新」「ページネーション表示更新」「ログ一覧描画」に分け、上部と下部のページネーションへ同じ `pageData` を反映する。
  - ページネーション操作は上下どちらのボタンでも `state.historyPage` を更新し、1 回の `renderHistory()` で両方を同期する。
  - 空履歴時は統計と上下ページネーションの disabled / 表示文言を一貫させる。
- `chrome-extension/styles.css`
  - 現在ページセクション向けに、パネル見出し行・開閉アイコン・開閉アニメーションを追加する。
  - 履歴統計の横並びテキスト行と、上下ページネーションを含む履歴ヘッダー/フッターのレイアウトを定義する。
  - モバイル幅では、統計行とページネーションが縦積みへ自然に崩れるようにする。

## Interface / Behavior
- 新規 public API は追加しない。変更対象は popup DOM とそのローカル state のみ。
- 開閉対象は「現在のページ」詳細全体で、少なくともタイトルと URL をまとめて隠す。
- 上下のページネーション表示は同一値を共有し、`1 / N`、prev disabled、next disabled が常に一致する。
- 表示範囲は現行ロジックを維持し、`0 - 0` または `start - end` の形式を継続する。

## Test Plan
- `npm run lint` と `npm test` を実行し、HTML/CSS/JS の静的検証と既存ユーティリティ回帰を確認する。
- popup の DOM テストを追加できない場合は、手動確認項目として次を report に残す。
  - 初期表示で現在ページセクションが閉じている。
  - トグル操作で `aria-expanded` と詳細表示が切り替わる。
  - 履歴が 11 件以上あるとき、上下どちらの next / prev でも同じページ番号・表示範囲・一覧内容に更新される。
  - 履歴 0 件時に統計と上下ページネーションが破綻しない。

## Assumptions
- エクスパンション化の対象は記録タブ内の「現在のページ」カードであり、入力フォーム側ではない。
- 「以前のように横並び」は、3 つの要素を 1 行の情報帯として並べる意味で解釈する。
- 上下ページネーションの文言とボタン名は既存のまま使い、見た目と配置のみ増やす。
